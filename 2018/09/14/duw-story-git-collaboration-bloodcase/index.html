熊本🐻 -> 怼刊协同血案-本地污染后需重新clone复本

Easy Bear 熊咩咩
2018-09-14
tech

背景

  • 本地修改仓库DUW分支上的文件 73w_draft.md 后
  • 将修改 git push 到du4proto DUW分支 被 rejected

  • 查看分支情况后，猜测，这是由于俺和其他怼员在各自的本地，对同一份文件 73w_draft.md 的相同lines作出修改，导致了协同事故



尝试

  • 用 git fetch + git merge 处理 non-fast-forward 错误
  • 终端反馈 cannot-merge

众怼

  • 求助 [42h][S]怼刊协同窘境-git push和git pull均被reject,git fetch + git merge处理后can not merge,需要手动修改冲突的地方吗？
  • 回复 @leilayanhui 在克隆 DUW 分支时, 是否使用 wiki 里 本地如何预防孤子分支合并 的方法

解决

  • 参考 本地如何预防孤子分支合并-万一本地副本被污染的解决方案
  • 将本地文件夹判定为 被污染文件
  • 重新clone新复本

后续

  • 本次血案诱发原因为，两位怼员在同一时间内，在本地对相同文件进行修改
  • 如何规避？ 诱发任务 -> 如何避免协同血案
