[42h][S]怼刊协同窘境-git push和git pull均被reject,git fetch + git merge处理后can not merge,需要手动修改冲突的地方吗？

(优先级别标定-> XXXL|XXL|XL|L|M|S 参考: TEE-style)

背景

~ 阐述 记录/问题/事件/… 发生的背景

  • 本地修改仓库DUW分支上的文件 73w_draft.md 后
  • 将修改 git push 到du4proto DUW分支 被 rejected
  • 试图用git pull 取回du4proto DUW分支更新 也被 rejected

分析

~ 先给出自己的态度以及尝试

  • 猜测，这是由于俺和其他怼员在各自的本地，对同一份文件 73w_draft.md 的相同lines作出修改，导致了协同事故

尝试

~ 尝试的过程/现象/…收集

  • 根据终端反馈，整理出努力的关键词

    • non-fast-forward
    • git fetch + git merge
    • merge conflicts

  • non-fast-forward

    • 根据 Dealing with non-fast-forward errors - User Documentation
    • 提示 You can fix this by fetching and merging the changes made on the remote branch with the changes that you have made locally:
      • git fetch origin [分支名]# Fetches updates made to an online repository`
      • git merge origin [分支名]:YOUR_BRANCH_NAME # Merges updates made online with your local work
    • 尝试 git fetch 和 git merge
  • git fetch + git merge
    • 远程库分支为:DUW;本地分支为:master
    • 终端输入 git fetch origin DUW 和 git merge origin DUW:master
    • 终端反馈
  • merge conflicts
    • 根据 Resolving a merge conflict using the command line - User Documentation
    • 提示 You can resolve merge conflicts using the command line and a text editor.
      • if you and another person both edited the file styleguide.md on the same lines in different branches of the same Git repository, you’ll get a merge conflict error when you try to merge these branches. You must resolve this merge conflict with a new commit before you can merge these branches.
      • 这段话是说，当冲突发生在同一个文件的修改时，必须先手动解决冲突，再进行新的提交commit
      • Open your favorite text editor, such as Atom, and navigate to the file that has merge conflicts.To see the beginning of the merge conflict in your file, search the file for the conflict marker <<<<<<<. When you open the file in your text editor, you’ll see the changes from the HEAD or base branch after the line <<<<<<< HEAD.
      • 这段话是说，编辑器会自动显示有冲突的地方，有 conflict marker <<<<<<<
  • conflict marker
    • 根据上述，似乎有些editor能够自动识别conflict，但俺的sublime text3 不能
    • 考虑安装plugin Zeeker/sublime-GitConflictResolver: A little plugin to help you resolving this nasty conflicts., 作为conflict marker
    • 但是又担心方向错了

问题

  • 现在俺的方向是
    • 手动修改目标文件冲突的地方
    • 下载 conflict marker
  • 请问
    • 手动修改的方向对吗？
    • 怎样才能让俺拿回du4proto DUW分支的更新，同时把俺本地的修改推送到du4proto DUW分支呢？

有效回复

后续方案

  • 暂定解决方案
    • 参考 本地如何预防孤子分支合并-万一本地副本被污染的解决方案

    • 重新clone新复本
